# Voice-Native Design Package

This package is the reviewable design output for:

- `WI-570` Frontend moodboard for the voice-native Starlog UI
- `WI-571` Translate the moodboard into component tokens and implementation guidance
- `WI-572` Voice-native interaction spec for PWA, mobile, and desktop helper surfaces

Files:

- `VOICE_NATIVE_MOODBOARD.md` — visual direction, references, and atmosphere
- `VOICE_NATIVE_TOKENS.md` — implementation-ready typography, color, spacing, and motion tokens
- `VOICE_NATIVE_SURFACE_SPEC.md` — surface-specific interaction rules for PWA, mobile, and desktop helper

Design intent:

- Chat and voice are the primary operating model.
- Surfaces should feel like mission control for a personal knowledge system, not a generic SaaS dashboard.
- PWA, mobile, and desktop helper share one language, but each emphasizes different actions.
